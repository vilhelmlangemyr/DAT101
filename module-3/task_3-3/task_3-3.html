1. Discussion Questions: 1. What is the difference between invoking and creating a function? Invoking a function means calling it to execute its code, while creating a function involves defining its structure and behavior without executing it. 2. Explain the syntax regarding semicolons and code blocks. Semicolons are used to terminate statements in JavaScript. Code blocks are defined using curly braces {} and group multiple statements together. 3. What happens when we try to store values from a function when it doesn't return any? If a function doesn't return a value, attempting to store its result will result in `undefined` being assigned to the variable. 4. How do we separate parameters from each other when creating a function? Parameters are separated by commas in the function definition, e.g., `function myFunction(param1, param2) { }`. 5. Explain how parameter names are related during the invocation and creation of a function. Parameter names in the function definition correspond to the arguments passed during invocation, allowing the function to access those values. 6. Explain why a parameter is undefined when a function call is missing a parameter. If a function is called without providing a value for a parameter, that parameter is assigned the value `undefined` by default. 7. Explain what happens when we use recursive functions. Recursive functions call themselves to solve a problem. They must have a base case to prevent infinite recursion, which can lead to stack overflow errors.
